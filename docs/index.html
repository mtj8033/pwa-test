Progressive Web App Example

Source
  • Provides an offline Todo list PWA utilizing:
    • Service Workers
    • Web App Manifests
    • IndexedDB
    • other common PWA APIs
  • Utilizes Workbox to easily setup our Service Worker to do things like:
    • Pre-cache files
    • Setup runtime caching rules based off of defined routes and specific caching strategies
    • Add background syncing functionality
Things to try:
  • Add tasks while offline/online and come back online
    • You have to actually go offline, can't just use the checkbox in Network tab of dev tools
    • Save icon means it was successfully posted to the mock backend being used
    • In progress icon means it hasn't been posted yet
    • When coming back online you can verify a http POST call was executed but the icon won't actually change
  • Reload the page and verify locally created tasks still show up
  • Install the application using the button at the top and try out the desktop/mobile experience
    • On iOS you should see a prompt to go through the share menu as they don't have full support yet
